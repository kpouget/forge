# Core Notifications Changelog

## 2026-06-24 - Notification System Improvements

### Enhanced Features
- **Streamlined Notification Mechanism**: Improved notification workflow with better error handling and fallbacks
- **FAILURE_REVIEW Integration**: Automatically includes FAILURE_REVIEW files generated by agentic agents
- **Custom Notification Support**: Allows projects to generate custom notifications via `NOTIFICATION.html` and `NOTIFICATION.md`

### New Notification Sources
- **Agent-Generated Reports**: Includes `failure_analysis_report.html` and `config_review.html` when available
- **FAILURE_REVIEW Files**: Auto-generated analysis summaries from on-failure agent in notification content
- **Custom Project Notifications**: 
  - `NOTIFICATION.html` - Custom HTML notification content for GitHub
  - `NOTIFICATION.md` - Custom Markdown notification content for Slack

### Improved Workflow
- **Smart Content Detection**: Automatically discovers and includes relevant analysis files
- **Graceful Fallbacks**: Falls back to legacy FAILURES file when no custom notifications exist
- **Better Formatting**: Enhanced message formatting with proper quoting and structure

### Notification Structure
```
000__ci_metadata/notifications/
├── 001__CONFIG_REVIEW_*.txt     # Auto-generated by agentic agents
├── 090__FAILURE_REVIEW_*.txt     # Auto-generated by agentic agents
└── *.txt                        # Other notification files
```

### Benefits
- **Richer Notifications**: More comprehensive failure analysis in CI notifications
- **Automated Analysis**: Agent-generated insights included automatically  
- **Project Customization**: Projects can provide custom notification content
- **Better User Experience**: More informative and actionable notification messages

### Files Modified
- `projects/core/notifications/send.py` - Enhanced notification content discovery and formatting
